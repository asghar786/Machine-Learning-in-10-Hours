1. Executive Summary
This document provides a complete technical and educational blueprint for "Machine Learning in 10 Hours" — an online certification course platform modeled after EdX. The system spans a React 19.2 frontend, a Laravel 12 API backend, and a MySQL 8 database, built API-first to support future mobile applications.
The course is structured into 5 sessions of 2 hours each, covering the full spectrum of Machine Learning from foundational mathematics to production deployment. Each learner receives a unique, randomized exercise set (50% variation) ensuring academic integrity and personalized learning. On successful completion, a verifiable digital certificate is issued.
Document Scope
• Course design: 5 sessions, learning objectives, exercises, assessments
• System architecture: API-first design, database schema, authentication
• Frontend specification: React 19.2 component hierarchy, state management, UI/UX
• Backend specification: Laravel 12 API endpoints, Eloquent models, queue jobs
• Certificate system: PDF generation, QR verification, blockchain-ready design
• Deployment & DevOps: Docker, CI/CD, environment configuration
2. Course Overview & Structure
2.1 Course Philosophy
The course is designed on three core principles: Learn by Doing (60% practical, 40% theory), Progressive Complexity (each session builds on the last), and Real-World Relevance (all examples use real datasets — Titanic, MNIST, Boston Housing, IMDB, and custom tabular data).

Mobile app readiness: React Native or Flutter can consume the same API
Third-party integrations: partner LMS systems can integrate via API keys
Independent frontend/backend deployment and scaling
Full Postman / OpenAPI (Swagger) documentation from day one

7.3 State Management Strategy
The app uses a layered state strategy to keep components clean and predictable:
TanStack Query v5: all server state (courses, sessions, submissions, user data). Auto-caches, background refetches, stale-while-revalidate.
Zustand v5: client-side global state only — auth token, user role, UI theme preference, sidebar open/closed.
React 19 Actions: form submissions and mutations. No manual loading state needed — useActionState handles pending/error/success automatically.
URL state (React Router v7): current session number, filter parameters, pagination — stored in URL so pages are shareable and bookmarkable.
7.4 Authentication Flow
User submits login form → React 19 Action calls POST /auth/login
On success, Sanctum token stored in Zustand (in-memory) + httpOnly cookie via Sanctum SPA mode
TanStack Query attaches token to all subsequent requests via global queryClient default
Route guards check Zustand auth state; protected routes redirect to /login if unauthenticated
On logout: revoke token via API, clear Zustand store, invalidate all queries, redirect to /

8. Certificate System
8.1 Generation Pipeline
Certificate generation is triggered asynchronously via a Laravel Queue job after a student passes the final quiz (score >= 70%). The pipeline:
Quiz submitted → POST /quiz/submit → QuizController validates answers → stores QuizAttempt with score
If score >= 70%: dispatch GenerateCertificateJob via Laravel Queue (Redis driver)
Job runs: creates certificates record with UUID v4, renders PDF template, stores to S3/local storage
Job completes: fires CertificateIssued event → sends congratulations email with PDF attachment
Frontend polls GET /certificates/my every 5s after quiz pass using TanStack Query until certificate appears
8.2 PDF Template Specification
The certificate PDF is generated using Laravel DomPDF or Snappy (wkhtmltopdf). Template elements:
A4 landscape format (297mm × 210mm)
Header: Course logo on left, platform name on right, decorative border
Body: "This certifies that [NAME] has successfully completed [COURSE]" in serif font
Details: Date, final score percentage, course duration (10 hours)
Footer: Instructor signature, platform seal, QR code (generated via PHP QR Code library)
UUID printed in small text below QR for manual verification
8.3 Verification URL
https://ml10hours.com/certificates/{uuid}
This public URL returns the certificate data as JSON (for API consumers) and renders the verification page for browsers. Certificate validity is checked against the certificates table — revoked_at NULL means valid.

9.4 Security Considerations
All API routes behind Laravel Sanctum — tokens hashed in DB, expire after 30 days of inactivity
Rate limiting: 60 req/min on public endpoints, 10 req/min on /auth/* to prevent brute force
File uploads validated: only .ipynb and .zip allowed, max 10MB, stored with UUID filename (not original)
SQL injection prevention: all DB interactions via Eloquent ORM — no raw queries except for analytics aggregates, all parameterized
CORS configured for the frontend domain only via config/cors.php
Secrets in .env: never committed; production uses AWS Secrets Manager or Laravel Vapor secrets
